02 — LỢI ÍCH THEO VAI TRÒ

Mỗi vai trò
một hành động cụ thể

Không chung chung. Chọn vai trò của bạn để xem hướng dẫn thiết thực nhất.

💉

Chống Injection & XSS

Dùng Parameterized Query / ORM — không bao giờ nối chuỗi SQL/NoSQL. Validate và escape HTML output chống XSS. Dùng bleach (Python) hoặc DOMPurify + Helmet CSP (Node.js).

CASE 1 — NGHIÊM TRỌNG · RCE
📁

File Upload An toàn

Kiểm tra Magic Bytes — không chỉ extension. Đặt tên file bằng UUID ngẫu nhiên. Lưu ngoài webroot, thư mục no-execute. Dùng python-magic hoặc file-type (npm) để verify.

CASE 2 — NGHIÊM TRỌNG · Webshell RCE
🔐

Phân quyền & IDOR

Luôn verify user_id == owner_id tại Backend — KHÔNG chỉ ẩn ở Frontend. Account Lockout sau N lần đăng nhập sai. JWT: khai báo bắt buộc algorithms=['HS256']. Token lưu HTTPOnly Cookie.

CASE 3 — CAO · Broken Auth
🔑

Secret & Dữ liệu Nhạy cảm

KHÔNG hardcode API key / password / token trong source code. Dùng .env + thêm vào .gitignore. API Response: lọc field qua Serializer (Django/DRF), Pydantic (FastAPI) hoặc .select() (Mongoose). Mask PII trong log.

CASE 4 — Sensitive Data Exposure
⚡

API Logic & Rate-limit

Chống Mass Assignment: destructure hoặc Allow-list field — không dùng thẳng req.body. Bắt buộc Rate-limiting mọi API public bằng express-rate-limit / flask-limiter. Chặn replay OTP/coupon bằng nonce.

CASE 5 — Logic Flaw · DoS
🚫

SSRF & Deserialization

CẤM tuyệt đối eval(), exec(), pickle.loads() với user input — dùng json.loads() / JSON.parse(). SSRF: Allow-list domain, chặn Private IP (10.x, 127.0.0.1, 169.254.169.254).

CASE 6 — NGHIÊM TRỌNG · RCE
🔒

Đóng Cổng Quản Trị

Đóng hoàn toàn SSH (22), RDP (3389), Docker API (2376), Redis (6379), Zookeeper (2181) ra Internet. Chỉ cho phép kết nối qua VPN nội bộ hoặc Bastion Host được xác thực. Dùng iptables/nftables để chặn Egress bất thường.

NHÓM 3 — HẠ TẦNG · NGHIÊM TRỌNG
☁️

Bảo mật Cloud & S3

Kiểm tra và tắt Public Read/Write trên S3/GCS/Azure Blob ngay lập tức. IAM Cloud: Least Privilege — không cấp quyền Admin cho service không cần. Bật CSPM tự động phát hiện cấu hình sai.

NHÓM 3 — CLOUD · Lộ dữ liệu
🩹

Patch CVE Kịp thời

Cập nhật OS, Middleware, thư viện ưu tiên CVE mức Critical/High trong vòng 72h. Theo dõi Log4j, OpenSSL, Struts, PHP — các component RCE phổ biến nhất. Dùng npm audit / pip-audit tự động trong CI.

NHÓM 3 — CVE · RCE
🛡️

Security Headers & TLS

Cấu hình bắt buộc trên Nginx/Apache/IIS: Content-Security-Policy, HSTS, X-Frame-Options: DENY, X-Content-Type-Options: nosniff. Vô hiệu hoá TLS 1.0/1.1 và Weak Cipher Suite.

NHÓM 4 — CONFIG · HTTPS
🔍

File Integrity & Threat Hunting

FIM (File Integrity Monitoring) phát hiện thay đổi bất thường trên binary hệ thống, webroot, cron job. Giám sát tiến trình lạ chạy quyền root, kernel module không rõ nguồn gốc (BPFDoor). Tắt hoàn toàn Debug mode & Directory Listing trên Production.

NHÓM 4 — Backdoor · FIM
🔑

Đổi Mật khẩu Mặc định

Đổi ngay mật khẩu mặc định của tất cả dịch vụ: OS, Database (MySQL root, PostgreSQL postgres), Grafana (admin/admin), Kibana, Router/Switch. Kiểm tra Setuid binary tránh leo thang đặc quyền (LPE). App kết nối DB không dùng tài khoản root/SA.

NHÓM 1 — IAM · LPE
📊

SIEM Tập trung & Alert

Thu log từ mọi server, firewall, WAF về SIEM tập trung (ELK, Splunk, QRadar). Cấu hình alert tự động: rà quét mật khẩu brute-force, thay đổi rule firewall bất thường, đăng nhập ngoài giờ từ IP không thuộc danh sách. Retention log tối thiểu 90 ngày.

NHÓM 4 — SIEM · Giám sát
🔭

CSPM & Port Scan Định kỳ

Cloud Security Posture Management tự động phát hiện S3 Bucket public, IAM role thừa quyền, Security Group mở rộng. Port scan định kỳ (nmap -sV) toàn bộ dải IP public — phát hiện service "quên đóng" sau mỗi deployment.

NHÓM 3 — CLOUD · Audit
🔄

DevSecOps CI/CD Pipeline

Tích hợp SAST (Semgrep, SonarQube — phân tích mã tĩnh) và DAST (OWASP ZAP — kiểm tra động) vào pipeline CI/CD. Build thất bại nếu phát hiện CVE Critical. Secret scanning tự động ngăn commit chứa key/token.

DEVSECOOPS · CI/CD
🔐

Audit SSO, MFA & Token

Audit định kỳ hệ thống SSO (Single Sign-On) và cấu hình MFA toàn tổ chức. Quét token / API key bị lộ trên GitHub/GitLab public repository bằng GitLeaks, TruffleHog. Review quyền tài khoản privileged mỗi quý.

NHÓM 1 — IAM · Token Leak
🎯

Pentest & Vulnerability Assessment

Lập kế hoạch Vulnerability Assessment định kỳ theo chuẩn OWASP Top 10. Phân loại lỗ hổng theo mức độ (Critical/High/Medium/Low) và SLA khắc phục. Báo cáo kèm PoC (Proof of Concept) để Dev/SysAdmin tái hiện và fix.

PENTEST · VA · OWASP
🤖

AI trong Bảo mật — Giới hạn

ĐƯỢC PHÉP: Dùng AI rà soát mã tĩnh, phân tích luồng dữ liệu, phát hiện Injection/Logic Flaw sớm trong review. NGHIÊM CẤM: Dùng AI tấn công (mô phỏng hoặc thực tế) vào hệ thống đang có khách hàng thật hoặc môi trường Production.

04 — SECURE CODING THỰC HÀNH

Code mẫu
❌ Sai vs ✅ Đúng

Đối chiếu trực tiếp Python và Node.js — áp dụng ngay vào dự án thực tế

🐍 Python
⚙️ Node.js
Python (Flask / Django / FastAPI)
❌ SQL Injection — KHÔNG làm thế này
# String concat — hacker inject được ngay cursor.execute("SELECT * FROM users WHERE username = '" + username + "'")
✅ Parameterized Query — Luôn làm thế này
# Parameterized — DB tự escape, an toàn hoàn toàn cursor.execute("SELECT * FROM users WHERE username = %s", (username,))
❌ IDOR — KHÔNG check ownership
Order.objects.get(id=request.GET['order_id'])
✅ IDOR Fixed — Luôn kèm điều kiện user
Order.objects.get(id=request.GET['order_id'], user=request.user)
❌ Hardcode Secret — KHÔNG bao giờ
SECRET_KEY = "abc123hardcoded" # Commit lên git → lộ ngay
✅ Env Variable — Đây là cách đúng
import os SECRET_KEY = os.environ.get('SECRET_KEY') # Từ .env
Node.js (Express / Mongoose)
❌ NoSQL Injection — KHÔNG truyền thẳng object
// Hacker gửi: {"username": {"$ne": null}} Model.find({ username: req.body.username })
✅ Ép kiểu + mongo-sanitize
import sanitize from 'mongo-sanitize' Model.find({ username: String(sanitize(req.body.username)) })
❌ Mass Assignment — Nguy hiểm
// Hacker gửi is_admin=true trong body await User.findByIdAndUpdate(id, req.body)
✅ Allow-list — Chỉ cho phép field cụ thể
const { name, email } = req.body // Destructuring await User.findByIdAndUpdate(id, { name, email })
❌ Deserialization nguy hiểm
eval(req.body.code) // RCE ngay lập tức
✅ Chỉ dùng JSON.parse an toàn
const data = JSON.parse(req.body.payload) // An toàn

🛡 Quy tắc 3 KHÔNG
Áp dụng cho mọi Developer

1
KHÔNG tin tưởng đầu vào
Mọi dữ liệu từ Body · Query · Params · Headers · File đều phải Validate và Sanitize trước khi xử lý.
2
KHÔNG dùng quyền cao nhất
App kết nối DB không dùng root/SA. Dịch vụ OS không chạy quyền root — tránh leo thang đặc quyền LPE.
3
KHÔNG nối chuỗi trực tiếp
Tuyệt đối cấm nối chuỗi với SQL · OS Command · Path/Directory. Luôn dùng thư viện chuẩn.

Hoàn toàn được. Tài liệu được xây dựng theo vai trò cụ thể — bạn chỉ cần đọc phần của mình (Dev / SysAdmin / InfoSec). Mỗi hướng dẫn đều có code mẫu cụ thể, không lý thuyết chung chung.
